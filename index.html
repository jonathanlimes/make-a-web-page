THE WEB


The Web was created in the 1990s, and is now currently estimated to hold about 30 billion pages. It is basically a interwoven network of computers that communicate with each other.

The Internet is almost synonymous with the Web. It is a network of documents that are stored in servers. The internet is able to house all sorts of documents, such as plain text, HTML, links (references to other documents on the web), images, and videos.

How users commonly access them is through a browser, like Google Chrome.

How does a browser access documents on servers through the Internet? A browser communicates with a server using a medium of communication that we call HTML (HyperText Markup Language).


HTML


HTML (HyperText Markup Language) is made up of plain text (the actual content you see on the web page), as well as markup (how the content is visually presented on the web page - the elements of a web page). HTML code adopts a tree-structure, with bigger branches splitting into smaller sub-branches, these sub-branches further splitting into more minute sub-branches.

As mentioned, a HTML document has elements: parts that come together to make a web page. Not all elements are visible, such as the 'head' component of the HTML code. All visible elements are actually rectangular or box-like, even if they may appear as other shapes. For example, a circle that appears on the web page is actually a box element. A whole web page is made up of a sum of boxes arranged in different ways.

To present content with the elements we want, such as italicizing each header just like this web page, we use tags. Most tags need to be closed with a corresponding closing tag. Otherwise, the tags will apply to the rest of the text, and your content will be presented in ways that you don't actually want.

There are two types of HTML elements: inline and block. Inline elements format content within the same segment of the page. Block elements create a separate segment, or a 'box' in which content is housed. An example of a block element are the separate paragraphs you find on this web page.

There are also attributes, which allow you to provide additional instructions within some tags, such as to provide a link to another page.

There are tons of HTML tags, far more than we can remember at any given point of time. A simple Google search will usually give you the tags you need. A good reference document can be found here!










CSS


CSS (Cascading Style Sheets) is used extensively to complement HTML in determining how a web page looks. Think of HTML as the foundation of a house, and CSS are the styles and colors that go into influencing how the house eventually looks. CSS allows you to use specific syntax and rules to change how content looks on a page.

CSS avoids the repetition that HTML so frequently commits. For example, if you want your h1 headers to be all blue, you don't have to code every single header in HTML to be blue. You designate in CSS that all h1 elements should have a specific color. Or perhaps font family, and size as well.

The important part about CSS is 'Cascading': This relies on an important concept in programming called inheritance. Using CSS, we cascade from higher, more general, overarching rules to smaller, more specific rules.






THE BOX MODEL


Within every box, there are four elements: Margin, Border, Padding, and Content. Most people who don't know web development don't know that there are three other layers other than the content layer that they see on the page. Each of these layers can be configured for their dimensions and appearance.



SIZING ISSUES


You will run into sizing issues. Sizing issues occur when your elements do not fit neatly into the page as represented by the browser window. They may sometimes be too big or too small, or they end up overlapping with other elements on the page.

There are two ways to deal with sizing issues:
Set sizes not in terms of pixels (px), but percentages.
Set 'box-sizing' attribute to 'border-box' for every element.




FLEX-BOXING


Elements are block - they will appear just like how an image appears via 'In line with text' in Microsoft Word. In other words, they will appear on their own unique line, making the page look pretty ugly, and difficult to control. If you use 'display:flex;' as a rule for every div element, you will let them appear flexibly side by side!



MENTAL MODELS FOR A PROGRAMMER


Computers are said to be 'stupid', because they interpret all instructions literally. You cannot provide them vague pointers and expect them to infer their orders like humans usually do. They operate on a different dimension of knowledge. However, if you provide clear and structured instructions to a computer, it can act as a powerful tool.

Programming requires systematic thinking and mental discipline. In the case of HTML, it is important to close all tags if they aren't void tags. Programmers need to develop good coding habits such as closing tags.

It's okay not to remember everything. There are too many details to keep everything in your head, and that is perfectly fine. Just look anything up on the Internet if you forget anything. It is your greatest common resource.

Avoid repetition. Simpler is better! Programmers should try to aim for less clutter, less repetition, and more elegance and simplicity in code. The clearer and more efficient it is, the better you and other programmers can work with it.

Code, Test, Refine. Learn to deconstruct website mockups like a Front-End Developer: Look for natural boxes, and look for repeated styles and semantic elements. Write your HTML code, and apply your styles on CSS, going from biggest to smallest. And then fix the small, specific things to continue perfecting your page design!
